## 一、部署架构概览

本系统部署在 K8s 集群内，分为两个 Namespace：

| Namespace | 组件 |
|-----------|------|
| `intel-system` | 管理 API、管理前端、采集器、调度器、MySQL、Milvus、Redis、Prometheus、Grafana |
| `llm-serving` | DeepSeek（或其他 LLM）+ Dify |

---

## 二、前置条件

### 2.1 基础设施要求

| 资源 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 16 核 | 32 核 |
| 内存 | 32 GB | 64 GB |
| GPU（LLM 节点） | 1× A100/H100 | 2× A100 |
| 存储 | 500 GB | 1 TB |
| K8s 版本 | 1.24 | 1.28+ |

### 2.2 软件依赖

```bash
# 检查版本

```

### 2.3 Dify 部署（先决条件）

本系统依赖 Dify Community Edition 作为 Workflow 编排引擎，需提前部署：

```bash

helm repo add dify https://charts.dify.ai
helm install dify dify/dify \
  --namespace intel-system \
  --create-namespace \
  --set global.storageClass=standard
```

## 三、部署步骤

### Step 1：克隆项目

```bash
git clone <repo-url> security-intel-agent
cd security-intel-agent
```

### Step 2：配置生产 values

```bash
cp helm/values.yaml helm/values-prod.yaml
vim helm/values-prod.yaml
```

关键配置项：

```yaml
# 修改为你的镜像仓库地址
imageRegistry: "registry.internal.company.com"

# 修改为正确的域名
ingress:
  hosts:
    adminWeb: "intel-admin.your-company.com"
    adminApi: "intel-api.your-company.com"

# 存储类（根据 K8s 集群实际情况）
mysql:
  storageClass: "ceph-rbd"   # 或 standard, gp2 等
milvus:
  storageClass: "ceph-rbd"
```

### Step 3：创建 K8s Secrets

编辑 `helm/templates/secrets.yaml`，将所有 `REPLACE_WITH_*` 占位符替换为真实值，然后：

```bash
# 方法1：kubectl apply（secrets.yaml 中填写真实值，注意安全）
kubectl apply -f helm/templates/secrets.yaml

# 方法2：命令行创建（更安全，避免密文落地）
kubectl create namespace intel-system

kubectl create secret generic mysql-credentials \
  -n intel-system \
  --from-literal=username=intel_admin \
  --from-literal=password='YOUR_MYSQL_PASSWORD'

kubectl create secret generic llm-credentials \
  -n intel-system \
  --from-literal=api-key='YOUR_LLM_API_KEY'

kubectl create secret generic wecom-bot-credentials \
  -n intel-system \
  --from-literal=webhook-url='https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY'

kubectl create secret generic search-api-credentials \
  -n intel-system \
  --from-literal=api-key='YOUR_360_SEARCH_KEY'

kubectl create secret generic smtp-credentials \
  -n intel-system \
  --from-literal=host='mail.company.com' \
  --from-literal=port='587' \
  --from-literal=username='intel-agent@company.com' \
  --from-literal=password='YOUR_SMTP_PASSWORD'

kubectl create secret generic dify-api-credentials \
  -n intel-system \
  --from-literal=api-key='YOUR_DIFY_API_KEY'
```

### Step 4：构建并推送 Docker 镜像

参见 [编译手册](01-编译手册.md) 第三节。

### Step 5：Helm 部署

```bash
# 验证配置（dry-run）
helm upgrade --install intel-agent helm/ \
  -n intel-system \
  -f helm/values.yaml \
  -f helm/values-prod.yaml \
  --dry-run --debug

# 正式部署
helm upgrade --install intel-agent helm/ \
  -n intel-system \
  --create-namespace \
  -f helm/values.yaml \
  -f helm/values-prod.yaml \
  --wait --timeout 10m
```

### Step 6：初始化数据库

```bash
# 等待 MySQL Pod 就绪
kubectl wait --for=condition=Ready pod -l app=mysql-primary \
  -n intel-system --timeout=120s

# 获取 MySQL Pod 名称
MYSQL_POD=$(kubectl get pods -n intel-system -l app=mysql-primary \
  -o jsonpath='{.items[0].metadata.name}')

# 获取密码
MYSQL_PWD=$(kubectl get secret mysql-credentials -n intel-system \
  -o jsonpath='{.data.password}' | base64 -d)

# 执行 DDL
kubectl exec -n intel-system "${MYSQL_POD}" -- \
  mysql -u root -p"${MYSQL_PWD}" < database/ddl.sql

# 执行种子数据
kubectl exec -n intel-system "${MYSQL_POD}" -- \
  mysql -u root -p"${MYSQL_PWD}" < database/seed.sql
```

### Step 7：导入 Dify Workflow

1. 访问 Dify 控制台：`http://dify.your-company.com`
2. 进入「工作流」→「导入 DSL」
3. 依次导入 `dify-workflows/` 目录下的所有 `.yml` 文件：
   - `wf-2-analysis-scoring.yml`
   - `wf-3-dedup.yml`
   - `wf-4-emergency.yml`
   - `wf-5-daily-report.yml`
   - `wf-6-weekly-report.yml`
   - `wf-7-monthly-report.yml`
   - `wf-8-periodic-report.yml`
   - `wf-9-push.yml`
   - `wf-10-feedback.yml`
   - `wf-11-health-check.yml`
4. 在 Dify 中为每个 Workflow 配置 LLM 模型提供商
5. 获取 Dify API Key 并更新 K8s Secret

### Step 8：配置 Dify 连接 LLM

1. 进入 Dify 「设置」→「模型提供商」
2. 添加 OpenAI 兼容接口：
   - API Base URL：`http://deepseek-server.llm-serving.svc.cluster.local:8000/v1`
   - API Key：LLM 服务的 Key
   - Model Name：`deepseek-chat`

### Step 9：初始化 Milvus Collection

```bash
# 进入 admin-api Pod
kubectl exec -it -n intel-system \
  $(kubectl get pods -n intel-system -l app=intel-admin-api -o jsonpath='{.items[0].metadata.name}') \
  -- python -c "
from pymilvus import connections, Collection, CollectionSchema, FieldSchema, DataType

connections.connect(host='milvus-standalone.intel-system.svc', port=19530)

fields = [
    FieldSchema('id', DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema('raw_intel_id', DataType.INT64),
    FieldSchema('embedding', DataType.FLOAT_VECTOR, dim=1536),
    FieldSchema('collected_date', DataType.VARCHAR, max_length=10),
]
schema = CollectionSchema(fields)
col = Collection('intel_vectors', schema)
col.create_index('embedding', {
    'index_type': 'IVF_FLAT', 'metric_type': 'COSINE', 'params': {'nlist': 128}
})
print('Milvus collection created successfully.')
"
```

### Step 10：验证部署

```bash
# 检查所有 Pod 状态
kubectl get pods -n intel-system

# 检查 LLM 连通性
python scripts/test-llm.py \
  --api-url http://deepseek-server.llm-serving.svc.cluster.local:8000/v1 \
  --api-key YOUR_LLM_KEY \
  --model deepseek-chat

# 访问管理控制台
# http://intel-admin.your-company.com
```

## 八、常见部署问题

### Q1: ImagePullBackOff

检查镜像仓库地址和认证：
```bash
kubectl create secret docker-registry regcred \
  -n intel-system \
  --docker-server=registry.internal.company.com \
  --docker-username=user \
  --docker-password=password
```

### Q2: MySQL Pod CrashLoopBackOff

检查存储类是否存在：
```bash
kubectl get storageclass
```

### Q3: Workflow 触发失败

检查 Dify API Key 是否正确：
```bash
kubectl get secret dify-api-credentials -n intel-system -o yaml
```
