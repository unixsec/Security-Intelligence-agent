## 一、概述

本项目包含以下需要编译/构建的组件：

| 组件 | 技术栈 | 构建产物 |
|------|--------|---------|
| RSS 采集器 | Python 3.11 | Docker 镜像 |
| 网页采集器 | Python 3.11 + Playwright | Docker 镜像 |
| 搜索采集器 | Python 3.11 | Docker 镜像 |
| 管理 API | Python 3.11 + FastAPI | Docker 镜像 |
| 管理前端 | Vue3 + Vite | 静态文件 + Docker 镜像 |

## 二、前置依赖

### 2.1 构建环境要求

| 工具 | 最低版本 | 说明 |
|------|---------|------|
| Docker | 20.10+ | 容器构建和推送 |
| Python | 3.11+ | 本地开发/测试 |
| Node.js | 20 LTS | 前端构建 |
| npm | 10+ | 前端依赖管理 |

### 2.2 安装 Python 依赖（本地开发）

```bash
# 采集器
cd collectors
python -m venv .venv
source .venv/bin/activate      # Linux/Mac
.venv\Scripts\activate         # Windows
pip install -r requirements.txt

# 管理 API
cd ../admin-api
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2.3 安装前端依赖

```bash
cd admin-web
npm install
```

## 三、构建 Docker 镜像

### 3.1 配置镜像仓库

将 `helm/values.yaml` 中的 `imageRegistry` 修改为企业私有镜像仓库地址：

```yaml
imageRegistry: "registry.internal.company.com"
```

登录镜像仓库：

```bash
docker login registry.internal.company.com
```

### 3.2 构建 RSS 采集器

```bash
cd collectors

# 构建镜像
docker build \
  -t registry.internal.company.com/intel/rss-collector:1.0 \
  -f rss_collector/Dockerfile \
  .

# 推送镜像
docker push registry.internal.company.com/intel/rss-collector:1.0
```

### 3.3 构建网页采集器

> 注意：网页采集器包含 Playwright + Chromium，镜像较大（约 1.5GB）

```bash
cd collectors

docker build \
  -t registry.internal.company.com/intel/web-collector:1.0 \
  -f web_collector/Dockerfile \
  .

docker push registry.internal.company.com/intel/web-collector:1.0
```

### 3.4 构建搜索采集器

```bash
cd collectors

docker build \
  -t registry.internal.company.com/intel/search-collector:1.0 \
  -f search_collector/Dockerfile \
  .

docker push registry.internal.company.com/intel/search-collector:1.0
```

### 3.5 构建管理 API

```bash
cd admin-api

docker build \
  -t registry.internal.company.com/intel/admin-api:1.0 \
  .

docker push registry.internal.company.com/intel/admin-api:1.0
```

### 3.6 构建管理前端

```bash
cd admin-web

# 多阶段构建（自动 npm build）
docker build \
  -t registry.internal.company.com/intel/admin-web:1.0 \
  .

docker push registry.internal.company.com/intel/admin-web:1.0
```

### 3.7 一键构建所有镜像

使用部署脚本（自动完成所有镜像的构建和推送）：

```bash
bash scripts/setup.sh
```

## 四、前端单独构建（不含 Docker）

如需将前端部署到已有 Nginx，可以单独构建静态文件：

```bash
cd admin-web

# 开发模式预览
npm run dev

# 生产构建
npm run build
# 构建产物位于：admin-web/dist/

# 构建产物部署到 Nginx
cp -r dist/* /usr/share/nginx/html/
```

## 七、构建常见问题

### Q1: Playwright 构建失败

Playwright 系统依赖较多，如构建报错，可尝试：
```bash
# 方案1：使用官方 Playwright 基础镜像
FROM mcr.microsoft.com/playwright/python:v1.40.0-jammy

# 方案2：跳过 Chromium，仅使用 requests（修改 web_collector/Dockerfile）
RUN playwright install chromium --with-deps
```

### Q2: 前端构建 node_modules 缺失

```bash
cd admin-web
rm -rf node_modules package-lock.json
npm install
```

### Q3: Docker 多平台构建（x86_64 + ARM）

K8s 节点为 ARM 架构时：
```bash
docker buildx create --use
docker buildx build --platform linux/amd64,linux/arm64 \
  -t registry.internal.company.com/intel/admin-api:1.0 \
  --push .
```
